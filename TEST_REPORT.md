## 4. App Bugs Found

### Confirmed Bugs: None New
The 46 existing failures are **stale test data**, not application bugs. The application code handles all edge cases tested.

### Observations (Not Bugs, But Notable)

1. **In-memory session Map has no eviction** — `const sessions = new Map()` grows unboundedly. Each session stores full conversation history. No TTL, no max size. This is a production memory leak risk under sustained usage.

2. **`listGLs()` returns GLs from mapping file regardless of data availability** — The dashboard could show GL options that have zero data for the selected week. Not a crash, but confusing UX.

3. **`appendToWeeklyFindings()` uses case-sensitive regex for GL headers** — If a GL is detected as "pc" but the header was written as "## PC", a duplicate section won't be created (the regex uses `/i`). But if someone manually adds "## Pc", it could create duplicates. Low risk with current GL names.

4. **Express body-parser logs stack trace on malformed JSON** — When sending invalid JSON to POST endpoints, Express's body-parser prints the full `SyntaxError` stack trace to `stderr`. The error is handled correctly (returns 400), but the console output leaks internal paths.
